# Plan: Stop previous sound on rapid re-trigger

## Problem
When a ball hits multiple blocks in quick succession, the same sound overlaps, creating cacophony.

## Solution
In `arkanoid/sound.py`, modify `_play()` to call `sound.stop()` before `sound.play()`. This stops any currently-playing instance of the same `pygame.mixer.Sound` object before starting a new one.
