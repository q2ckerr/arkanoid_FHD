# Plan: Silver Brick Durability by Level

## Research Result

Silver bricks in the original Arkanoid require increasing hits to destroy as the game progresses.

**Formula**: `hits = 2 + (round_no - 1) // 8`

| Rounds | Hits |
|--------|------|
| 1–8    | 2    |
| 9–16   | 3    |
| 17–24  | 4    |
| 25–32  | 5    |

### Sources
- StrategyWiki: "Silver bricks take more than one hit to destroy. In the beginning, they only require two to destroy, but the number of hits it takes to remove them increases by one every eight stages."
- NES Manual: "2 times - 1st to 8th round, 3 times - 9th to 16th round, 4 times - 17th to 24th round, 5 times - 25th to 32nd round"
- PrimeTime Amusements: "the amount required to destroy them increases by 1 every 8 levels"
- GameFAQs: "Silver 50 x # of level, increase by 1 every 8 levels"

## Current Code Status

In `arkanoid/sprites/brick.py`, silver bricks have `_destroy_after = 2` (hardcoded). The `round_no` is already passed to the `Brick.__init__` constructor but is only used for score calculation.

## Implementation

### File: `arkanoid/sprites/brick.py`

In `Brick.__init__`, change the silver brick `_destroy_after` from hardcoded `2` to formula-based:

```python
if brick_colour == BrickColour.silver:
    self._destroy_after = 2 + (round_no - 1) // 8
```

This is a one-line change. The `round_no` parameter is already available in the constructor.
